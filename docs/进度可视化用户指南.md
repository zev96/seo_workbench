# 进度可视化功能用户指南

## 功能介绍

在生成Word文档时，系统现在会显示一个美观的进度对话框，让您实时了解生成进度，并可以随时取消操作。

## 界面说明

### 进度对话框元素

```
┌─────────────────────────────────────────┐
│  ✓ 正在生成文档                          │
│                                          │
│  正在处理: 25/100 (25%)                  │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                          │
│  正在生成第 25 个文档（混排模式）...      │
│                                          │
│                              [  取消  ]   │
└─────────────────────────────────────────┘
```

**说明**：
- **标题**：显示当前任务名称
- **状态行**：显示当前进度（已完成/总数，百分比）
- **进度条**：视觉化显示进度
- **详细信息**：当前正在执行的具体操作
- **取消按钮**：随时中止生成任务

## 使用步骤

### 1. 准备数据
- 在工作区导入或编辑您的内容
- 确保数据格式正确

### 2. 开始生成
1. 点击工具栏的"开始生成 Word"按钮
2. 选择生成模式（按行生成/随机混排）
3. 设置生成数量（混排模式）
4. 选择保存目录

### 3. 查看进度
- 进度对话框自动弹出
- 实时显示：
  - 当前进度（X/Y）
  - 完成百分比
  - 正在处理的文档信息

### 4. 完成或取消

#### 正常完成
- 进度条到达100%
- 显示"✓ 生成完成"
- 右下角弹出成功通知
- 点击"关闭"按钮退出

#### 中途取消
- 点击"取消"按钮
- 系统安全停止生成
- 已生成的文档会保留
- 显示取消提示

## 使用场景

### 场景1：批量生成文档
**情况**：需要生成100个文档

**操作**：
1. 设置生成数量为100
2. 点击"开始生成"
3. 选择保存目录
4. 观察进度对话框
5. 等待完成

**优势**：
- 知道已完成多少
- 估算剩余时间
- 可以做其他事情

### 场景2：测试生成效果
**情况**：想先生成几个看看效果

**操作**：
1. 设置生成数量为5
2. 开始生成
3. 看到前3个已完成
4. 如果效果不好，点击"取消"
5. 修改设置后重新生成

**优势**：
- 快速测试
- 避免浪费时间
- 灵活调整

### 场景3：大批量生成
**情况**：需要生成500+文档

**操作**：
1. 开始生成
2. 查看进度对话框确认正在运行
3. 最小化窗口
4. 做其他工作
5. 不时查看进度
6. 完成后收到通知

**优势**：
- 不阻塞工作
- 随时了解进度
- 自动通知完成

## 常见问题

### Q1: 可以在生成过程中关闭对话框吗？
**A**: 不建议。对话框是模态的，关闭会取消生成。如需取消，请点击"取消"按钮。

### Q2: 点击取消后已生成的文档会丢失吗？
**A**: 不会。已保存的文档都会保留在您选择的目录中。

### Q3: 进度条卡住不动怎么办？
**A**: 
- 检查是否有文件被其他程序占用
- 查看详细信息了解当前操作
- 如果长时间无响应，可以点击取消
- 查看日志文件了解详细错误

### Q4: 生成速度会受影响吗？
**A**: 不会。进度显示是轻量级的，不影响生成速度。

### Q5: 能否后台运行？
**A**: 当前版本不支持。对话框需要保持打开状态。未来版本会支持后台运行。

## 技术说明

### 更新频率
- 每完成一个文档更新一次进度
- 避免频繁更新影响性能
- 保证信息的实时性

### 资源占用
- **CPU**: 几乎无额外占用
- **内存**: 增加约2-3MB
- **磁盘**: 无影响

### 兼容性
- Windows 10/11: 完美支持
- 其他系统: 功能完整，样式可能略有差异

## 提示和技巧

### 💡 提示1: 预估时间
虽然没有显示剩余时间，但您可以：
- 记录生成10个文档的时间
- 计算平均每个文档的时间
- 估算总时间 = 平均时间 × 剩余数量

### 💡 提示2: 批量生成优化
生成大量文档时：
- 先测试生成5-10个
- 确认效果满意后再大批量生成
- 避免生成后发现问题需要重新生成

### 💡 提示3: 利用等待时间
生成过程中可以：
- 准备下一批数据
- 查看已生成的文档
- 做其他工作（界面不会冻结）

### 💡 提示4: 错误排查
如果遇到问题：
1. 查看详细信息了解卡在哪一步
2. 点击取消停止任务
3. 检查日志文件（logs目录）
4. 修复问题后重新生成
